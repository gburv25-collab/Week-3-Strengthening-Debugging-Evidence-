VS Code Debugging Evidence

Definition

  • Debugging is the process of identifying, investigating and correcting defects in code.
  • Breakpoint is a marker that pauses the program at a specific line so that you can inspect what is happening while the program is running.
  • Step-through debugging allows you to move through code one line at a time so that you can observe how the program behaves.
  • Variable inspection allows you to check the values stored in variables while the program is paused.
  • IntelliSense is a VS Code feature that provides coding support such as suggestions, code completion and prompts while you write or edit code.
  • Validation checks whether input is suitable before the program uses it.
  • Testing checks whether the program behaves as expected using different inputs or conditions.
  • Software quality refers to how reliable, accurate, usable, robust, maintainable and readable a software product is.

Why it matters

Assignment 1 requires you to demonstrate and explain debugging and validation methods using small code evidence.

It is not enough to show corrected code. Stronger evidence explains how the issue was identified, what debugging method was used, what was observed, what improvement was made and how this improved quality.

Using VS Code debugging tools can help you produce stronger evidence because you can show the process of investigating code rather than only presenting the final version. Testing after changes also provides evidence that the correction works across normal, invalid and boundary inputs.

Practical Code Notice

The practical activity uses a Cinema Ticket Cost Checker. This is practice code for learning how to use the VS Code debugger.

It is not the code provided for Assignment 1. You should use the process from this activity to help you independently investigate and explain your own Assignment 1 evidence.

The practice code may not crash immediately. Instead, it may produce unexpected results because user input is stored as text rather than as numbers. This makes it useful for exploring how debugging can help identify issues that are not always obvious from syntax alone.

Practice Code

# Cinema Ticket Cost Checker
# Practice code for learning how to use VS Code debugging tools

adult_ticket_price = 9
child_ticket_price = 5

adult_tickets = input("Enter number of adult tickets: ")
child_tickets = input("Enter number of child tickets: ")

adult_total = adult_ticket_price * adult_tickets
child_total = child_ticket_price * child_tickets

total = adult_total + child_total

print("Total cost: £" + total)

Practical Activity: VS Code Debugging Evidence Workshop

Step 1: Create the Python file

Open VS Code. Create a new file and save it as week3_cinema_debugging.py. Make sure the file ends in .py.

Step 2: Paste in the practice code

Paste the Cinema Ticket Cost Checker code into the file and save it.

Step 3: Run the code normally first

  1. Right-click inside the file.
  2. Select Run Python File in Terminal.
  3. Enter adult tickets: 2 and child tickets: 1.
QuestionYour notes
What is the program trying to calculate?
What should the expected total be?
What actual output did the program produce?
Does the behaviour suggest a type, logic or output-format issue?
Did the program crash or produce an incorrect result?

Step 4: Notice VS Code editor support

Check whether VS Code gives any visual clues, such as syntax highlighting, bracket matching, IntelliSense suggestions, underlined issues, warnings, messages or terminal output.

PromptYour notes
What editor support did you notice?
Did VS Code highlight anything useful?
Did IntelliSense provide any useful suggestions?
Did the terminal output give any useful information?
How could this support general error finding?

Step 5: Set a breakpoint

Find this line:

adult_total = adult_ticket_price * adult_tickets

Set a breakpoint by clicking in the margin to the left of the line number. A red dot should appear. If execution reaches this line, the debugger will pause before the line runs.

QuestionYour notes
Which line did you place the breakpoint on?
Why is this a useful place to pause the program?

Step 6: Start debugging

  1. Make sure the Python file is open.
  2. Press F5.
  3. If VS Code asks what you want to debug, choose the Python file/debug option shown.
  4. Enter the same test values.
  5. The program should pause at the breakpoint if execution reaches that line.
QuestionYour notes
Did the program pause at the breakpoint?
What line is about to run?
What values have already been entered?
